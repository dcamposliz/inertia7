PROJECT: PREDICTING THE STOCK MARKET



TIME-SERIES ANALYSIS OF THE S&P 500 STOCK INDEX WITH R

Project Summary

Abstract


This project focuses on finding the best statistical-learning model to predict future values for the S&P 500 Stock Index.


Understanding the S&P 500 Stock Index is highly-relevant in understanding the health of the U.S. economy as it is highly-correlated (statistically) with other U.S. economic indicators such as other stock indices (e.g. the NASDAQ Stock Index and the NYSE Stock Exchange), the U.S. Housing Price Index, and the U.S. Gross Domestic Product (U.S. GDP).


Time-series analysis is a basic concept within the field of statistical-learning, which is appropriate for the analysis of the S&P 500 Stock Index.


For this project we leverage the horse-power of R and deliver, where appropriate, gorgeous data visualizations using d3.js and plotly.js


We encourage you to try replicating this project and make your own contributions! You can fork this project on GitHub.

1.

Load Packages

First, we want to load the appropriate packages into our R environment.


For this we use the required() method and include the package names as arguments. Make sure to first install the packages using the install.packages("") method if you haven't done so already.


require(data.table)

2.

Get Data

Collecting Data


Now we collect our data. We want to use reliable sources of complete and accurate data. We collected 21 years (1995-2015) of S&P 500 Stock Index data at a monthly frequency (a total of 252 observations) from Yahoo Finance. You can do the same too.



Cleaning Data


We cleaned our data manually using Google Sheets. For this we made sure that all fields contained data (i.e. no missing values) and that the headers (i.e. column names) were labeled appropriately. This can be done programmatically with R, but this is outside of the scope of this project.


We also included a bunch of other variables (for exploratory analysis) such as the NASDAQ Stock Index, the NYSE Stock Exchange, the U.S. Housing Price Index, and the U.S. Gross Domestic Product. Here is our data in file data.csv, which is ready to be used with R.



Loading Data


Then we must include our data set within our working R environment. For this we use:


dataMaster <- read.csv(" ...{PATH_TO}... /data.csv")
attach(dataMaster)

Now we can call our S&P 500 Stock Index data by typing dataMaster$sp_500 into our terminal.

3.

Do exploratory analysis

Now we want to get a feel for our data to be able to make inferences as to which models may be appropriate for our prediction. For this, we plot our data and diagnose for trend, seasonality, heteroskedasticity, and stationarity. We go over these concepts in further detail in this section.



Creating time-series data object


Our S&P 500 Stock Index data is in the form of time-series; this means that our data exists over a continuous time interval with equal spacing between every two consecutive measurements. In R we are able to create time-series objects for our data vectors using the ts() method. For this, we select the vector we would like to use as the first argument, and tune the start and freq (frequency) parameters. Then we output the time-series data to the terminal by calling our newly-created time-series object.


sp_500 <- ts(dataMaster$sp_500, start=c(1995, 1), freq=12)

Then we output the time-series data to the terminal by calling our newly-created time-series object. Notice the nicely formatted data broken down into years and their respective months.


sp_500





PLOTTING OUR TIME SERIES


Plotting the data is arguably the most critical step in the exploratory analysis phase. This enables us to make inferences about important components of the time-series data, such as trend, seasonality, heteroskedasticity, and stationarity. Here is a quick summary of each:


  • Trend: we say that a dataset has a trend when it has either a long-term increase or decrease.

  • Seasonality: we say that a dataset has seasonality when it has patterns that repeat over known, fixed periods of time (e.g. monthly, quarterly, yearly).

  • Heteroskedasticity: we say that a data is heteroskedastic when its variability is not constant (i.e. its variance increases or decreases as a function of the explanatory variable).

  • Stationarity: a stochastic process is called stationary if the mean and variance are constant (i.e. their joint distribution does not change over time).

We can plot our sp_500 data object in R using the plot() method.


plot.ts(sp_500)

With a little more 'wrangling' we can plot our data using plotly.js, a browser-based data-visualization library written in JavaScript.





Decomposing our time-series


Now we will decompose our time-series into its different components: seasonal component, the trend, and residuals.


plot(stl(sp_500, s.window = "periodic"), main = "Decomposition for S&P 500")


Most of what we are seeing we had already identified but here we can see there are moments of decreasing trend but overall we can see that the time series is gradually increasing as time progresses. We can see there is seasonal components which is expected in financial and annual data. We will see later if they have a large influence in our model building.




DIAGNOSING THE ACF AND PACF OF OUR TIME-SERIES


These models are used under the assumption of stationarity, we are not going to go into detail with respect to the theory since there already is a lot of documentation on that! But these models can be estimated with what are called autocorrelation and partial autocorrelation functions.

The autocorrelation function and the partial autocorrelation function. The autocorrelation function tells you of the correlation of different data points at different time lags (lag is defined as an equal time interval h where we have a time t so the lagged value would be t-h).

The next plots are in the astsa package, when wanting to deduce the orders of the models we will be using for the time series, the ACF and PACF plots are a helpful guide for this.


acf2(sp_500)


These plots tell us that the model is not stationary and that we would should do a transformation to adjust this. We take a difference to get rid of the trend present in our data.




TRANSFORMING OUR DATA TO ADJUST FOR NON-STATIONARITY


Because our time series and ACF/PACF plots indicate non-stationarity we have to make the necessary transformation to achieve stationarity. A way to make a time series object stationary is to find the difference between consecutive values, this allows for the mean to stabilize thereby making the time series object stationary which we then could estimate the ARIMA model from.


diff1 <- diff(sp_500)

Next we plot our transformed time-series.


plot.ts(diff1, main="Time Series Plot of First Diff")


The plot of the first difference seems to suggest stationarity so the next step is to check the ACF/PACF plots to see if it is stationary and if we can estimate a model from them!




DIAGNOSING THE ACF AND PACF OF OUR TRANSFORMED TIME-SERIES


As previously stated once we have stationary we use the ACF/PACF plots to deduce what model we have but we come in this mode with a difference parameter equal to one (i.e. d = 1) since we are estimating using the differenced time series.


acf2(diff1)


From these plots we deduce that our model best fits an MA(1) since there is the ACF cuts off at one significant lag and the PACf shows geometric decay. Recall we are examining the differenced time series so we have to use the combined model ARIMA (autoregressive integrated moving average), thus our model is ARIMA(0, 1, 1) and we can continue forecasting.

4.

Build models

We decided to go with the Box-Jenkins approach and build an ARIMA model while also using other models to compare which produced the best outputs. WIthin the forecast package there is a function called auto.arima that yields the best model based on Akaike-Information-Criterion (AIC). The AIC is a measurement of quality for a model compared to others. So it one of many model selection in time series, although we do warn against just using this model without testing other models out since it has been known not to produce the best output.


But the model we received from both the differenced data set and the original data set yielded the same result, and after testing we reached the same conclusion; that an ARIMA(0,1,1) model is our best fit.


So we created a data frame with out the year 2015 and fit our model on this data to see how well the model predicts compared to the real data.


Here’s the code for removing and storing the actual values of 2015 in a separate data frame:


dataMaster_TS <- dataMaster[-c(1:240),]
dataMaster_2014df <- dataMaster[-c(241:252),]
dataMaster_TR <- ts(dataMaster_2014df, start = c(1995, 1), freq = 12)
sp500_TR <- dataMaster_TR[, 'sp_500']
sp500_TR

Due to the best model being ARIMA(0,1,1) we fitted the appropriate model using the Arima function in the forecast package:


fit <- Arima(sp500_TR, order = c(0,1,1), include.drift=TRUE)

Our next step was running residual diagnostics to ensure our residuals were white noise under our initial assumptions! So we used the tsdisplay function to check our residuals:


tsdisplay(residuals(fit))

We also decided to create a histogram of the residuals to give people another visualization of the normally distributed residuals!


hist(residuals(fit), prob = TRUE)
lines(density(residuals(fit))

Although not perfect we can see that the residuals do display a normal distribution with outliers explained by the financial crisis of 2007-2008. We made a more in depth analysis of these assumptions on the 2nd iteration of this project if you would like to explore this version it is readily available!

Thus we believe we have found a model that follows all our assumptions, and we can continue to forecasting with this model!

5.

Forecast

So with our model we proceeded to forecast using the forecast package (duh!). We found our best model and we’ll forecast the values for the S&P 500 for 2015. So we created a data frame where we removed the last 12 values from our sp_500 earlier, and created a separate data frame that contained these values. Again we would like to say that there is a more complex iteration of this with interactive plots using plotly, but the coding might be a bit complex for people doing any introductory time series project.


act_sp500_2015_ts <- ts(dataMaster_TS$sp_500, start=c(2015, 1), freq=12)
act_sp500_2015_ts

This should have a time series object for S&P 500 from 1995-2014 and another time series object with the values for 2015. Here we forecasted for these values then added the actual values to see how it compared.


plot(forecast(fit, h = 12), xlab = “Years”, ylab = “S&P Values”)
lines(act_sp500_2015_ts, col = “red”)



Although this plot does tell us a lot we can barely see the values so the beauty of R is that we can change our x and y limits to get a better view of the actual vs predicted values for 2015! We may be repeating ourselves but there is another version with ggplot on the second iteration of the project! Thus we input this to get a closer view.


plot(forecast(fit, h = 12), xlim = c(2015, 2016), ylim = c(1600, 2400), xlab = “Years”, ylab = “S&P Values”)
lines(act_sp500_2015_ts, col = "red")



Thus we can see the model gives a pretty good estimation for the forecast for the year 2015. It falls within the 95%/80% confidence intervals and follows the flow of the actual values, even though the actual values are really volatile!

6.

Derive Conclusions

Our final step was comparing our model to other models to see which has the lowest accuracy measurement criterion. As suggested by Rob J Hyndman, these tools are better at measuring which model is best fit, because different functions in R use different parameters to measure AIC. So AIC isn’t comparable for different models.


“In general, be careful comparing the AIC value between software packages as they may treat the constant term differently, and then the AIC values are not comparable. This is the most likely explanation here.” – Rob Hyndman Apr 12 at 23:39


(Important to note this was him responding to my question on Cross Validated section of the Stack Exchange website, we can’t stress enough the amount of information you can get there especially from the people who created most of these r packages!!!! - Raul)
